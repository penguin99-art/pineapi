> §1–§4 是客户需求的结构化；「性质判断」列与 §5 是我们的标注，非客户原文。

## 0. 系统全景

- **端一 · 门店智能终端（设备）**：完整门店业务终端（CRM/评估/产品/会员/报告/科普六大模块），端侧大模型离线推理是其中的 AI 切片。
- **端二 · 店员工作台（门店 Web / App）**：肠菌报告解读、售后智能问答；报告批量上传含 OCR。
- **端三 · 总店管理平台（总部后台）**：通过产品推荐权重影响各门店端侧模型的推荐排序。
- **跨端 · 意向客户智能触达**：精准图文推送、AI 主动外呼、全场景语音转写归档。

> 注意：端一与端二的 AI 能力（报告解读、问答）重叠——这是 UI 之分不是能力之分，AI 能力在盒子建一次、多端共用（`integration-plan.md` §2）。

## 1. 端一 · 门店智能终端（以设备版功能图为准）

| 模块 | 功能点 | 性质判断 |
| --- | --- | --- |
| 客户管理 | 客户建档、客户分类、客户标签、客户数据等 | 传统 CRM，业务应用层；AI 可选增值=自动打标签 |
| 评估系统 | 问卷评估、体检数据收集 | **AI 旗舰链路入口**（细化见 §1.1）；体检数据是新输入维度 |
| 产品相关 | 产品推送、产品介绍、配方菌株介绍 | 推送=AI 推荐链路；介绍类=静态内容（产品库资产渲染） |
| 会员 | 会员注册、线上/线下支付及确认、线下产品核销 | 交易业务，纯业务侧；**支付涉资质，不进盒子承诺范围** |
| 肠菌报告 | 报告上传、报告分析/对比、出具诊疗方案 | 解读=AI（与端二同一能力）；**对比=同客户多次报告纵向趋势**；**「出具诊疗方案」措辞触医疗红线，须改口径（§5#2）** |
| 科普知识 | 科普文章、百问百答 | 文章=内容运营；百问百答=RAG 问答**直面消费者**（护栏比店员版高一档） |

### 1.1 AI 评估推荐链路细化（原功能提取表，AI 线最细的需求口径）

端侧大模型为本端核心：盒子内置模型，离线本地推理，无需联网即可完成个性化产品推荐。

| 功能点 | 功能说明 | 优先级 |
| --- | --- | --- |
| 离线推理引擎 | 盒子内置模型，断网可推理；加载产品库、权重策略、回答规范库，无需云端即可输出推荐 | **P0 核心** |
| 评估数据输入 | 问卷答案结构化送入模型：基础信息、疾病史、家族病史、肠道症状、用药补充史、调理需求等字段 | **P0 核心** |
| 评估数据输出 | 输出：评估结论、个性化推荐、推荐理由（症状匹配+菌属匹配）、异常指标科普+知识点高亮、可存档结构化数据 | **P0 核心** |
| 个性化推荐生成 | 匹配产品及理由说明；关联：便秘/腹泻、疾病风险、营养缺乏、调理需求 | **P0 核心** |
| 店员人工调整 | 可替换/补充推荐产品，保留操作记录 | P1 |
| 推荐结果存档 | 问卷答案+AI 推荐+店员调整记录绑定客户档案 | **P0 核心** |

## 5. 待澄清清单（我们的标注，按影响排序）

**商务前置（M0 必须拍板）**：

1. **终端业务应用谁开发**（头号商务问题）：设备功能 2/3 是 CRM/会员/支付/核销/内容等传统软件——客户期望「买设备=整套 app」还是「盒子+AI 能力、app 由 SI/客户开发」？决定合同性质与报价，先于一切排期。
2. **「出具诊疗方案」措辞**：医疗执业用语，无证出具违法。须确认改「调理方案/健康管理建议」——红线，不可妥协。
3. **盒子算力档位/预算**：决定端侧模型尺寸与选型，一期选型的前置阻塞。

**范围与机制**：

4. **三端部署形态**：端二/端三/客户档案跑在哪（盒子/门店服务器/总部云），决定范围与报价边界。
5. **离线 vs 总部下发**：产品库/权重的下发格式、频率、断网期间用旧版的容忍度（方案预设 skill 包版本下发，需客户认可节奏）。
6. **体检数据收集**：范围（血常规？体脂？）、来源（手填/设备对接/照片）、格式——决定评估输入 schema 的 `checkup?` 块。
7. **肠菌报告输入格式**：检测机构出 PDF/图片/结构化数据？决定 OCR 技术选型与精度承诺。
8. **报告对比口径**：多次报告比什么（指标趋势/菌属变化）、展示形式。
9. **百问百答回答边界**：直面消费者，哪些问题拒答/转店员（用药、疾病判断类），客户圈定。

**盒外通道与责任**：

10. **支付通道**：接谁的通道、资质归属——明确盒外，写进边界。
11. **外呼通道归属**：电话线路/呼叫平台是云端 SaaS——客户已有采购、我们集成，还是期望全包？
12. **推送渠道**：微信公众号/小程序/短信/App push？账号主体与接口权限谁提供。
13. **合规审查责任**：健康数据+自动外呼+录音转写涉个保法、录音告知、医疗宣传边界——谁负责合规口径。
14. **知识库与规范库内容来源**：客户提供还是我们整理？反馈回流谁维护。
