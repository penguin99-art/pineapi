写 Pinea 文档（中文）时遵守两条：

1. **不用口语黑话**，用正式工程表述。用户逐处改掉的例子：白嫖→复用、赌模型→不依赖模型判断/自觉、裸奔→不开门锁、钦定姿势→标准部署方式、回打→回调、藏掉→对外隐藏、吃/吐 OpenAI 形状→输入/输出、长肉→代码量增长、串台→会话混淆、拍脑袋→以实测为准、翻车→风险、不用嘴说→不靠口头承诺、抓手→基础机制/支撑承诺。
2. **承诺纪律**：尚未落地/未进 Stable 矩阵的增量（如 T1.5 结构化保障、X-Pinea-Priority QoS 头），在方案/场景文档里必须标注"行业增量，落地并升档前不作为 contract-v1.0 承诺"，验收标准也按"升档前实测口径 / 升档后才可承诺 100%"分写。另：接口语义描述要精确（如 DELETE 不轻易写"幂等"，写明"重复调用不复活、不存在返回 404"）。

**Why:** 2026-06-11 用户对 14 个文件做了细粒度修订，全部是这两类——文档面向 SI/客户/商务，口语黑话不专业；把排期中的能力写成现行承诺会造成对外超卖。
**How to apply:** 起草或修改 pineapi / pinea-docs 任何文档时执行；引用排期能力时先核对它在 tob-overview §7 矩阵的档位再决定措辞。
